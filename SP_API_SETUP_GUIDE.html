Amazon SP-API Setup Guide

Connect Your Dashboard to Live Amazon Data

What this guide covers: Getting your Amazon Selling Partner API (SP-API) credentials so the GolfGen dashboard can pull live sales, orders, inventory, and financial data from your Seller Central account. Once set up, you'll run a single Python command on your Mac to refresh the dashboard with real data.

Part 1: Register as a Developer ~10 min

1
Go to Seller Central Developer Central

Log into sellercentral.amazon.com with your GolfGen seller account.

Navigate to: Apps & Services → Develop Apps (or go directly to Developer Console).

If you see "Register as a developer" — click it and complete the form.

2
Fill in the Developer Profile

Use your company details:

  • Organization name: GolfGen LLC (or Elite Global Brands LLC)
  • Data Protection Officer contact: Your email
  • Primary use case: "Building internal analytics dashboard"
  • API type: Select SP-API
3
Create a New App

After registering as a developer, click "Add new app client"

  • App name: GolfGen Dashboard
  • API Type: SP API
  • IAM ARN: You'll fill this in after Part 2 (skip for now if allowed, or create the IAM role first)

Once created, you'll see your LWA Client ID and LWA Client Secret. Copy these — you'll need them.

Part 2: Set Up AWS IAM ~15 min

Amazon requires API calls to be signed with AWS credentials. You need a free AWS account with a specific IAM role and user.

4
Create a Free AWS Account (if you don't have one)

Go to aws.amazon.com/free and create an account. The free tier is all you need — SP-API itself costs nothing.

5
Create an IAM Policy

In the AWS Console, go to IAM → Policies → Create Policy. Click the JSON tab and paste:

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "execute-api:Invoke",
            "Resource": "arn:aws:execute-api:*:*:*"
        }
    ]
}

Name it: SellingPartnerAPIPolicy

6
Create an IAM Role

Go to IAM → Roles → Create Role

  • Trusted entity: AWS Account → "This account"
  • Attach policy: Select SellingPartnerAPIPolicy from step 5
  • Role name: GolfGenSPAPIRole

After creating, click the role and copy the Role ARN — it looks like:
arn:aws:iam::123456789012:role/GolfGenSPAPIRole

7
Create an IAM User

Go to IAM → Users → Create User

  • User name: golfgen-sp-api
  • Access type: Programmatic access (Access key)
  • Permissions: Attach the SellingPartnerAPIPolicy directly, AND also add an inline policy to allow assuming the role:
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "sts:AssumeRole",
            "Resource": "arn:aws:iam::YOUR_ACCOUNT_ID:role/GolfGenSPAPIRole"
        }
    ]
}

After creating the user, go to Security credentials → Create access key. Copy your AWS Access Key ID and AWS Secret Access Key.

!
Important: Update Your App with the Role ARN

Go back to Seller Central → Developer Console → your app. Edit it and paste the IAM Role ARN from step 6 into the IAM ARN field.

Part 3: Authorize Your App ~5 min

8
Self-Authorize (since you're the seller AND the developer)

In Seller Central, go to: Apps & Services → Manage Apps

Find "GolfGen Dashboard" and click "Authorize".

This generates your Refresh Token. Copy it immediately — you won't be able to see it again.

Part 4: Configure & Run on Your Mac ~5 min

9
Install Python Dependencies

Open Terminal on your Mac and run:

# Install Python 3 if you don't have it
brew install python3

# Navigate to the dashboard folder
cd ~/Desktop/golfgen_amazon_dashboard

# Install required packages
pip3 install python-amazon-sp-api duckdb requests
10
Add Your Credentials

Copy the template and fill in your real credentials:

# Copy the template
cp config/credentials_template.json config/credentials.json

# Edit with your credentials
open -e config/credentials.json

Fill in every YOUR_... field with the values you collected in Parts 1-3.

!!
Never share credentials.json or commit it to git!

This file contains your secret keys. The .gitignore already excludes it, but be careful if you move the folder.

11
Run the Data Pipeline 
# Pull last 7 days of data
python3 scripts/amazon_sp_api.py

# Pull last 90 days (for initial backfill)
python3 scripts/amazon_sp_api.py 90

# Generate the dashboard HTML from real data
python3 scripts/generate_dashboard.py

After running, open amazon_dashboard_v3.html in your browser to see your live data.

12
Set Up Auto-Refresh (Optional)

To automatically pull fresh data every morning:

# Open Mac's cron editor
crontab -e

# Add this line (runs at 7 AM daily)
0 7 * * * cd ~/Desktop/golfgen_amazon_dashboard && python3 scripts/amazon_sp_api.py && python3 scripts/generate_dashboard.py

Credentials Checklist

Before running, make sure you have all 7 values:

  • LWA Client ID — from Developer Console → Your App
  • LWA Client Secret — from Developer Console → Your App
  • Refresh Token — from self-authorizing your app
  • AWS Access Key ID — from IAM User creation
  • AWS Secret Access Key — from IAM User creation
  • Role ARN — from IAM Role creation
  • Seller ID — from Seller Central → Account Info

Troubleshooting

Error: "Access to requested resource is denied"

Your app hasn't been authorized yet, or the Refresh Token is wrong. Re-do Step 8.

Error: "The security token included in the request is invalid"

AWS credentials are wrong. Double-check your Access Key, Secret Key, and Role ARN in credentials.json.

Error: "Report request failed"

Some reports (like Sales & Traffic) require your account to have Brand Registry. If you have Brand Registry, make sure you've selected the correct marketplace (US = ATVPDKIKX0DER).

Error: "python-amazon-sp-api not installed"

Run pip3 install python-amazon-sp-api again. If you have multiple Python versions, try python3 -m pip install python-amazon-sp-api.
